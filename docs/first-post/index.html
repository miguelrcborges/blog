My experience on remaking the blog on Hugo

It was confusing at first. I wasn’t being able to find how I would theme it without relying on a theme. Since I only was aware that styling was only available through themes, I searched how to make my own, where I found out the folder structure of making a theme coincides with the site’s.

The next weird thing was finding out how pages rendered. If you have a index.html in layouts it will override the generated one. Hugo, to generate the files, relies on 3 .html files:

  • layouts/_default/baseof.html - The base html for everypage.
  • layouts/_default/list.html - Processed on the index page.
  • layouts/_default/single.html - Processed on every file inside the content directory.

I had mention “_default” on every directory, but if you have markdown files inside different directories of content, you can specify different rules by adding a folder in layouts with the same folder name, instead of “_default”.

After figuring how it works, the experience has been nice.
